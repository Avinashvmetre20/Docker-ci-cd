Linux Commands

Files · Processes · Networking · Permissions · Archives

File & Directory
ls -la

List all files including hidden ones with permissions, size and date.

$ ls -la /home/user
cd <dir>

Change directory. cd .. goes up, cd ~ goes home, cd - goes back.

$ cd /var/log
mkdir -p <dir>

Create a directory and all missing parent directories at once.

$ mkdir -p projects/app/src
rm -rf <path>

⚠️ Force-delete files and folders recursively. This is irreversible!

$ rm -rf ./node_modules
cp -r <src> <dst>

Copy files or entire directories recursively.

$ cp -r ./src ./backup
mv <src> <dst>

Move or rename files and directories.

$ mv old_name.txt new_name.txt
find . -name "*.log"

Search for files recursively by name pattern, type, or size.

$ find /var -name "error.log"
pwd

Print the full path of the current working directory.

$ pwd → /home/user/projects
touch <file>

Create an empty file or update its last-modified timestamp.

$ touch index.html
ln -s <target> <link>

Create a symbolic (soft) link to a file or directory.

$ ln -s /usr/bin/python3 python
View & Edit Files
cat <file>

Print the full contents of a file to the terminal.

$ cat /etc/hosts
less <file>

Page through large files. Press q to quit, / to search inside.

$ less /var/log/syslog
tail -f <file>

Follow a file live — perfect for watching logs in real time.

$ tail -f /var/log/nginx/access.log
grep -rn "text" .

Recursively search text inside files with line numbers.

$ grep -rn "TODO" ./src
nano / vim <file>

Edit files in terminal. nano is beginner-friendly; vim is powerful.

$ nano /etc/nginx/nginx.conf
wc -l <file>

Count lines (-l), words (-w), or characters (-c) in a file.

$ wc -l app.log
sed -i 's/old/new/g'

Find and replace text in files. -i edits the file in-place.

$ sed -i 's/localhost/0.0.0.0/g' config.txt
Processes & System
ps aux

List all running processes with CPU and memory usage.

$ ps aux | grep nginx
kill -9 <PID>

Force-terminate a process by PID. Try kill PID first for graceful stop.

$ kill -9 12345
htop

Interactive real-time process and resource monitor.

$ htop
df -h

Show disk space usage for all filesystems in human-readable format.

$ df -h /
du -sh <dir>

Show total size of a directory or file.

$ du -sh ./node_modules
free -h

Display RAM and swap memory usage in human-readable units.

$ free -h
uname -a

Show kernel version, hostname, and architecture info.

$ uname -a
systemctl restart <svc>

Manage services: start, stop, restart, status, enable.

$ systemctl restart nginx
Permissions & Users
chmod 755 <file>

Set permissions. 755=owner rwx, others rx. Use +x to add execute.

$ chmod +x deploy.sh
chown user:group <f>

Change file owner and group. Add -R for recursive.

$ chown -R www-data:www-data /var/www
sudo <command>

Run a command with superuser (root) privileges.

$ sudo apt update && sudo apt upgrade
whoami / id

Show current username or full user/group identity.

$ id → uid=1000(user) gid=1000
Networking
curl -X GET <url>

Make HTTP requests. -I headers only, -d POST body, -o save to file.

$ curl -s https://api.ipify.org
wget <url>

Download a file from the internet.

$ wget https://example.com/file.zip
ping <host>

Test network reachability and measure latency to a host.

$ ping google.com
ss -tulnp

Show open ports and the processes listening on them.

$ ss -tulnp | grep 80
ssh user@host

Connect to a remote server securely via SSH.

$ ssh -i key.pem ubuntu@1.2.3.4
scp <src> user@h:<dst>

Securely copy files to/from a remote server over SSH.

$ scp app.tar.gz user@server:/tmp/
Archives & Compression
tar -czf out.tar.gz <dir>

Create a compressed gzip archive of a directory.

$ tar -czf backup.tar.gz ./data
tar -xzf file.tar.gz

Extract a .tar.gz archive. Add -C ./dir to extract into a folder.

$ tar -xzf release.tar.gz -C ./dist
unzip <file.zip>

Extract a .zip archive. Add -d ./output to choose destination.

$ unzip archive.zip -d ./output

Docker Commands

Images · Containers · Volumes · Networks · Compose

Images
docker pull <image>

Download an image from Docker Hub or a private registry.

$ docker pull nginx:latest
docker images

List all locally available images. Add -a to include intermediate layers.

$ docker images -a
docker build -t <tag> .

Build an image from the Dockerfile in the current directory.

$ docker build -t myapp:1.0 .
docker rmi <image>

Remove one or more local images. Use -f to force removal.

$ docker rmi myapp:old
docker tag <src> <dst>

Tag an image with a new name — required before pushing to a registry.

$ docker tag app user/app:v2
docker push <image>

Push a tagged image to Docker Hub or a registry.

$ docker push user/myapp:v2
Containers
docker run -d -p 80:80

Run a container in background, mapping host port to container port.

$ docker run -d -p 3000:3000 --name api myapp
docker ps

List running containers. Add -a to include stopped ones.

$ docker ps -a
docker stop / start <id>

Gracefully stop or start a container by name or ID.

$ docker stop api && docker start api
docker rm <container>

Remove a stopped container. Use -f to force-remove a running one.

$ docker rm -f old_container
docker exec -it <c> bash

Open an interactive shell inside a running container.

$ docker exec -it mydb bash
docker logs -f <c>

Stream live container logs. Add --tail 100 to limit output.

$ docker logs --tail 100 -f api
docker inspect <c>

Return detailed JSON metadata about a container or image.

$ docker inspect api | jq '.[0].NetworkSettings'
docker stats

Live CPU, RAM and network usage for all running containers.

$ docker stats --no-stream
docker cp <src> <c>:<dst>

Copy files between host and a container (both directions work).

$ docker cp config.json api:/app/
Volumes & Networks
docker volume create <v>

Create a named volume for persistent data storage.

$ docker volume create db_data
docker volume ls / prune

List volumes or remove all unused ones.

$ docker volume prune
docker network create <n>

Create a bridge network for inter-container communication.

$ docker network create app-net
docker network ls

List all Docker networks. Use inspect to see connected containers.

$ docker network inspect app-net
Docker Compose
docker compose up -d

Start all services in docker-compose.yml in the background.

$ docker compose up -d --build
docker compose down

Stop & remove containers and networks. Add -v to also remove volumes.

$ docker compose down -v
docker compose logs -f

Follow live logs from all or a specific Compose service.

$ docker compose logs -f web
docker compose ps

List status of containers in the Compose project.

$ docker compose ps
docker compose exec <svc>

Run a command or open a shell inside a running Compose service.

$ docker compose exec db psql -U admin
docker compose build

Rebuild images for services that have a build key defined.

$ docker compose build --no-cache
Cleanup
docker system prune -a

⚠️ Remove ALL unused images, containers, networks and build cache.

$ docker system prune -a --volumes
docker image prune -f

Remove dangling (untagged) images to free up disk space.

$ docker image prune -f

Git Commands

Commits · Branches · Remotes · History · Tags

Setup & Init
git init

Initialize a new Git repository in the current directory.

$ git init my-project
git clone <url>

Clone a remote repository to your local machine.

$ git clone git@github.com:user/repo.git
git config --global

Set global name, email, or default editor used across all repos.

$ git config --global user.email "you@mail.com"
Staging & Committing
git status

Show staged, unstaged and untracked files in the working tree.

$ git status -s
git add <file>

Stage files for commit. Use . to stage everything at once.

$ git add src/ package.json
git commit -m "msg"

Commit staged changes with a message. Use prefixes: feat:, fix:, docs:.

$ git commit -m "feat: add login page"
git commit --amend

Modify the most recent commit (message or staged files).

$ git commit --amend --no-edit
git diff

Show unstaged changes. Use --staged to see what's queued to commit.

$ git diff HEAD~1
git stash

Shelve uncommitted changes. Restore with git stash pop.

$ git stash push -m "WIP: feature"
Branches
git branch

List branches. -a includes remote, -d deletes, -m renames.

$ git branch -d feature/login
git checkout -b <br>

Create and switch to a new branch in one command.

$ git checkout -b feature/signup
git switch <branch>

Modern way to switch branches (Git 2.23+). Use -c to create new.

$ git switch main
git merge <branch>

Merge another branch into the current one. --no-ff preserves history.

$ git merge --no-ff feature/login
git rebase <branch>

Replay commits on top of another branch for clean linear history.

$ git rebase -i HEAD~3
git cherry-pick <hash>

Apply a specific commit from another branch onto the current one.

$ git cherry-pick a3f2c91
Remotes
git remote -v

List all remote connections with their URLs.

$ git remote add origin git@github.com:u/r.git
git fetch --all --prune

Download remote changes without merging. --prune cleans deleted branches.

$ git fetch --all --prune
git pull --rebase

Fetch and rebase (instead of merge) for a cleaner commit history.

$ git pull --rebase origin main
git push -u origin <br>

Push branch to remote and set it as the upstream tracking branch.

$ git push -u origin feature/api
git push --force-with-lease

Safe force push: aborts if someone else pushed changes in the meantime.

$ git push --force-with-lease origin main
History & Undoing
git log --oneline --graph

Compact visual commit history showing all branches as a graph.

$ git log --oneline --graph --all
git show <hash>

Show the diff and metadata of a specific commit.

$ git show HEAD~2
git reset --soft HEAD~1

Undo last commit but keep changes staged — safe to recommit.

$ git reset --soft HEAD~1
git reset --hard <hash>

⚠️ Discard all changes and reset to a commit. This is destructive!

$ git reset --hard origin/main
git revert <hash>

Create a new commit that reverses another. Safe for shared branches.

$ git revert a3f2c91
git reflog

Show full history of HEAD movements — use to recover "lost" commits.

$ git reflog | head -20
Tags & Advanced
git tag -a v1.0 -m "msg"

Create an annotated release tag at the current commit.

$ git tag -a v2.0.0 -m "Release 2.0"
git push --tags

Push all local tags to the remote repository.

$ git push origin --tags
git blame <file>

Show who last changed each line in a file and when.

$ git blame -L 10,30 src/app.js
git bisect

Binary-search commits to find which one introduced a bug.

$ git bisect start; git bisect bad; git bisect good v1.0
